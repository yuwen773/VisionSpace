# VisionSpace 功能路线图

> 生成日期：2026-04-04
> 基于当前项目状态分析，列出可实现的功能规划

## 项目现状概览

VisionSpace 是一个**云端智能图片管理与协作平台**，技术栈：

- **后端：** Spring Boot 3.5.8 + Java 21 + MyBatis Plus + Sa-Token + Spring AI Alibaba
- **前端：** Vue 3 + TypeScript + Ant Design Vue + Pinia + ECharts
- **部署：** Docker Compose（MySQL 8.0 + Redis 7 + Spring Boot）

### 已完成功能

| 模块 | 功能 |
|------|------|
| 用户系统 | 注册/登录/注销、VIP 兑换、个人中心 |
| 图片管理 | 上传（单张/批量/URL）、编辑、审核、颜色搜索、AI 扩图 |
| 空间协作 | 私有/团队空间、成员角色管理（查看者/编辑/管理员）、实时协作编辑 |
| 推荐系统 | 首页瀑布流推荐（热度/最新/质量/随机）、加权评分、多样性重排 |
| 行为分析 | 用户行为追踪（曝光/点击/浏览/点赞/收藏/下载/分享）、统计聚合 |
| 空间分析 | 6 种图表（使用量/分类/标签/大小/用户/排名） |
| AI Agent | 多工具智能体（图片搜索/生成/特征分析/质量评估/相似搜索）、SSE 流式对话 |
| 存储管理 | 多平台存储（COS/OSS/MinIO）、管理端配置 |
| 反馈系统 | 用户反馈提交、管理端处理 |
| 主题系统 | 双主题（深空暗黑 / 波普风-紫嫣） |
| 管理后台 | Dashboard、用户/图片/空间/存储/反馈管理 |

## 一、高优先级 — 补全已有但不完整的功能

### 1.1 VIP 兑换系统前端对接

- **现状：** 后端 `UserController` 已有 VIP 兑换接口，数据库有 `vipExpireTime`、`vipCode`、`vipNumber` 字段，DTO `VipExchangeRequest` 已定义
- **缺失：** 前端 `VipSection.vue` 标注 TODO，未对接真实接口
- **工作内容：**
  - 对接 VIP 兑换 API
  - 展示 VIP 状态、到期时间
  - 兑换码输入与验证交互
- **预估工作量：** 小（1-2 天）

### 1.2 WebP 预览图格式升级

- **现状：** 项目已升级至 Java 21，`PicturePreviewServiceImpl.java:63` 标注 TODO 等待切换 WebP
- **工作内容：**
  - 将预览图生成格式从 JPEG 切换为 WebP
  - 处理历史数据的兼容性（已有 JPEG 预览图）
  - 更新前端预览图展示逻辑
- **预估工作量：** 小（1 天）

### 1.3 推荐系统管理端手动评分

- **现状：** `HomeRecommendScoreCalculator.java:119` 预留了 10% 权重的手动评分，但无管理端入口
- **工作内容：**

  - 前端：管理端图片列表增加评分列，支持批量/单个打分
  - 对接推荐评分公式
- **预估工作量：** 中（3-4 天）

### 1.4 Agent 会话状态恢复

- **现状：** `ImageAgent.java:343` 标注 TODO，checkpoint 数据存储但未正确读取恢复
- **工作内容：**

  - 前端侧边栏历史会话点击恢复
- **预估工作量：** 中（3-5 天）

### 1.5 单元测试与集成测试

- **现状：** pom.xml 引入了 `spring-boot-starter-test`，但几乎无测试代码
- **工作内容：**

## 二、中优先级 — 新增功能

### 2.1 图片收藏夹/相册

- **功能描述：** 用户可创建收藏夹（公开/私密），收藏图片到不同分类
- **数据库设计：** 新增 `collection`（收藏夹）、`collection_picture`（收藏关系）表
- **工作内容：**
  - 后端：收藏夹 CRUD、图片收藏/取消、收藏夹详情
  - 前端：图片详情页收藏按钮、我的收藏页、收藏夹管理
- **预估工作量：** 中（3-5 天）

### 2.2 图片评论系统

- **功能描述：** 对图片进行评论、回复，支持楼中楼
- **数据库设计：** 新增 `picture_comment` 表（parentId 支持嵌套）
- **工作内容：**
  - 后端：评论 CRUD、楼中楼查询、评论数统计
  - 前端：图片详情页评论区、回复交互、评论排序
- **预估工作量：** 中（4-5 天）

### 2.3 通知系统

- **功能描述：** 审核结果通知、协作邀请、评论提醒、系统公告
- **数据库设计：** 新增 `notification` 表
- **工作内容：**
  - 后端：通知生成、已读标记、未读计数
  - 前端：通知铃铛组件、通知列表、实时推送（WebSocket）
- **预估工作量：** 中（4-6 天）

### 2.4 图片标签智能推荐

- **功能描述：** 上传图片时利用 AI 自动识别并推荐标签和分类
- **技术方案：** 调用 Spring AI Alibaba 多模态模型分析图片内容
- **工作内容：**
  - 后端：图片内容分析接口、标签推荐逻辑
  - 前端：上传页展示推荐标签、用户可确认/修改
- **预估工作量：** 中（3-5 天）

### 2.5 用户关注/粉丝

- **功能描述：** 关注其他用户，查看关注用户的最新图片动态
- **数据库设计：** 新增 `user_follow` 表
- **工作内容：**
  - 后端：关注/取关、关注列表、粉丝列表、动态 Feed
  - 前端：用户主页、关注按钮、关注动态页
- **预估工作量：** 中（4-5 天）

### 2.6 OAuth 第三方登录

- **功能描述：** 支持 GitHub / Gitee / 微信登录
- **技术方案：** Sa-Token OAuth2 集成（项目已使用 Sa-Token）
- **工作内容：**
  - 后端：OAuth 回调处理、账号绑定/自动注册
  - 前端：登录页第三方登录入口、绑定管理
- **预估工作量：** 中（4-6 天）

### 2.7 回收站功能

- **功能描述：** 删除的图片进入回收站，30 天内可恢复，超期自动清理
- **技术方案：** 利用现有逻辑删除 `isDelete` 字段，增加恢复接口和定时清理任务
- **工作内容：**
  - 后端：恢复接口、定时清理任务、回收站列表查询
  - 前端：回收站页面、恢复/永久删除操作
- **预估工作量：** 小（2-3 天）

## 三、较高复杂度 — 进阶功能

### 3.1 图片 OCR 文字识别

- **功能描述：** 上传图片自动提取文字内容，支持文字内容搜索图片
- **技术方案：** 调用阿里云 OCR API 或 Spring AI 多模态模型
- **数据库设计：** `picture` 表新增 `ocrText` 字段
- **预估工作量：** 大（5-8 天）

### 3.2 图片相似度去重

- **功能描述：** 上传时检测重复/相似图片，提示用户
- **技术方案：** 感知哈希（pHash）+ 汉明距离比对
- **预估工作量：** 大（5-7 天）

### 3.3 数据导出/报表

- **功能描述：** 空间分析数据导出为 Excel/PDF，支持自定义报表
- **技术方案：** Apache POI（Excel）+ iText（PDF）
- **预估工作量：** 中（3-5 天）

### 3.4 操作日志审计

- **功能描述：** 管理员查看系统关键操作日志（登录/删除/权限变更等）
- **数据库设计：** 新增 `operation_log` 表
- **技术方案：** AOP 注解 + 异步日志记录
- **预估工作量：** 中（3-4 天）

### 3.5 国际化 i18n

- **功能描述：** 支持中文/英文/日文等多语言切换
- **技术方案：** vue-i18n + 后端 MessageSource
- **预估工作量：** 大（7-10 天）

### 3.6 PWA 离线支持

- **功能描述：** 移动端可安装到桌面、支持离线浏览已加载内容
- **技术方案：** Service Worker + Workbox + manifest.json
- **预估工作量：** 中（3-5 天）
