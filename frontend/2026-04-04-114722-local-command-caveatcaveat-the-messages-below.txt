● 图片有两种处理方式：

  1. Markdown 正文中的图片 — 通过 v-html="renderedContent" 渲染。useMarkdown 把
  markdown 文本转成 HTML，图片会以 markdown 原生方式（![alt](url)）渲染成 <img>
  标签。但目前没有对 markdown 中的图片做特殊样式处理，渲染结果取决于 markdown
  渲染器的默认行为。
  2. images prop 传入的图片 — 以内联缩略图形式直接展示（第 10-18
  行）。这些图片：
    - 显示为 72x72 的缩略图网格
    - 点击触发 previewImage 事件（用于全屏预览）
    - 不是超链接形式

  总结：图片链接都是直接展示图片，不是超链接。images prop
  的图片有专门的缩略图样式和点击预览交互；markdown 正文中的图片则依赖 markdown
  渲染器的默认输出。

● 核心区别在 markdown 语法上：

  消息 1 用的是 [文字](url) — markdown 链接语法
  - 渲染为 <a href="...">水彩猫眼绘画</a>
  - 显示为可点击的蓝色文字超链接

  消息 2 用的是 ![文字](url) — markdown 图片语法
  - 渲染为 <img src="..." alt="图片1">
  - 直接内嵌显示图片

  但消息 2 有个问题：.markdown-body 内没有任何针对 img
  的样式约束。这些图片会以原始尺寸渲染（URL 中 h=350 意味着高度
  350px），在消息气泡 max-width: 680px 的容器里可能会：

  - 撑得很大，占据整个气泡宽度
  - 多张图片纵向堆叠，消息变得很长
  - 没有圆角、间距、hover 等样式，视觉上很粗糙

  而 images prop
  的缩略图（72x72、圆角、可点击预览）是有专门样式处理的，但这两条消息都没传
  images prop，所以那个缩略图区域不会出现。

  总结：消息 1 是整洁的文字链接列表；消息 2
  会直接嵌入大图，但缺少样式控制，显示效果可能不理想。
